# AFSIM + AI 智能体专家能力路线

先说结论：读完代码和跑通一个 EOIR demo，不会自动成为专家。它会给你一套正确的工程骨架。真正的专家能力来自“能独立建模、能设计实验、能解释结果、能证明 AI 系统可靠”，而不是会调用一个大模型。

## 一、专家能力的五个支柱

| 支柱 | 达标表现 | 本项目现状 |
|---|---|---|
| AFSIM 建模 | 能从任务问题建立平台、传感器、处理器、通信、武器、行为和事件模型 | EOIR 入门已覆盖，其余待扩展 |
| AI 工程 | 能设计 Schema、RAG、Agent、工具调用、离线模型和评测 | 已建立工程骨架 |
| 仿真实验 | 能提出假设、控制变量、批量运行、统计分析和解释不确定性 | 第 21 步开始覆盖 |
| 软件工程 | 能做版本、测试、配置、审计、故障定位和离线交付 | 已覆盖基础，仍需模块化和 CI |
| 领域判断 | 能判断模型参数与任务口径是否合理，知道结果能否支持决策 | 必须结合真实课题持续训练 |

## 二、六阶段学习路线

### 阶段 1：AFSIM 语言与运行机制

学习材料优先使用本机正版安装内容：

```text
D:\AFsim\AFSim\afsim-2.9.0-win64\documentation\index.html
D:\AFsim\AFSim\afsim-2.9.0-win64\training\user
D:\AFsim\AFSim\afsim-2.9.0-win64\demos
```

需要掌握：

- type 与 instance 的区别；
- include、平台、side、route、execute、script 的执行关系；
- sensor、processor、comm、weapon 与平台的组合方式；
- simulation time、随机数、事件调度与状态变化；
- `.evt` 文本事件与 `.aer` 二进制事件管道的用途。

毕业作品：不用 AI，独立修改一个 demo，解释每个关键字，预测事件输出，再用 `.evt` 验证预测。

### 阶段 2：结构化想定编译器

把自然语言问题分成三层：

```text
任务意图层：侦察什么、何时、成功标准是什么
领域模型层：平台、传感器、目标、关系和约束
AFSIM 实现层：include、type、instance、script 和 output
```

需要掌握：

- 为什么模型只生成 JSON IR，不直接生成 AFSIM；
- JSON Schema、组件白名单和跨字段约束的边界；
- 确定性渲染、静态检查和错误定位；
- IR 版本号、向后兼容和迁移。

毕业作品：新增一种参数变化仍能稳定编译的 EOIR 想定，并为非法坐标、非法组件和错误动作顺序编写测试。

### 阶段 3：本地 RAG 与 Agent

需要掌握：

- 文档切分、元数据、术语归一化、召回率与来源追踪；
- 何时关键词检索足够，何时才需要本地 embedding；
- draft、validate、repair、simulate、analyze 各节点的职责；
- MCP 是接口协议，Skill 是操作规范，两者不会提升模型推理能力；
- 模型不可用、超时、格式错误和越界请求的安全降级。

毕业作品：给一个错误需求和一个超范围需求，解释 Agent 在哪一层拒绝，证明没有规则兜底制造假成功。

### 阶段 4：仿真结果与实验方法

需要掌握：

- 任务指标、模型内部指标和计算运行指标的区别；
- 探测概率、首次发现时间、目标覆盖率、航迹连续性等指标；
- 单次运行不能代表统计规律；
- 随机种子、重复实验、均值、方差、置信区间与异常值；
- 参数扫描、灵敏度分析、消融实验和基线对比。

毕业作品：选择高度、距离或传感器参数作为自变量，批量运行多组仿真，说明它如何影响首次发现时间和探测成功率。报告必须能复现。

### 阶段 5：多领域想定扩展

建议顺序：

1. 雷达搜索与跟踪：与 EOIR 同属传感器链路，迁移成本最低。
2. 通信网络：学习链路、路由、消息与网络约束。
3. 武器交战：学习决策、发射、制导、毁伤和交战结果。
4. 电子战：学习探测、干扰、对抗和效果评估。

每新增一类都必须拥有：组件来源、IR Schema、领域校验、RAG 知识、正负评测集、AFSIM 实跑、事件效果门禁和人工审核。不能因为模型能写出文本就标记为支持。

毕业作品：完成雷达领域端到端闭环，至少包含一个成功案例、一个参数边界案例和一个必须拒绝的非法案例。

### 阶段 6：工程与研究级系统

需要掌握：

- 配置、数据、组件、提示词、模型和 AFSIM 版本的联合追踪；
- 并发任务隔离、资源限制、超时、重试和幂等性；
- 离线网络边界、数据分级、访问控制与审计；
- 模型升级 A/B 测试和回归门禁；
- 人在回路的审批职责，而不是只放一个“确认”按钮；
- 不确定性表达：哪些是事实、模型假设、推断和未知。

毕业作品：让另一名使用者只按文档在新机器上复现一个完整实验，并得到同一范围内的指标和可核验报告。

## 三、建议实践课题

每个课题都必须先不用 AI 做出可信基线，再让 AI 自动生成或修改：

1. EOIR 高度/距离对首次发现时间的影响。
2. 雷达搜索周期与航迹连续性的关系。
3. 通信受限条件下消息到达率与任务效果。
4. 不同交战规则对武器消耗和任务成功率的影响。
5. 干扰条件下传感器/通信性能退化与恢复策略。

## 四、专家级自测问题

你应当能够不查答案解释：

- 为什么 `mission.exe` 返回 0 不能证明想定正确？
- 为什么 RAG 文档不能替代组件白名单和代码校验？
- 为什么 0.5B 在固定示例上输出 JSON 仍不能上线？
- 如何区分模型错误、IR 错误、渲染错误、AFSIM 组件错误和任务口径错误？
- 如何设计一个实验判断 7B 是否真的优于 0.5B？
- 随机仿真为什么要保存种子和重复次数？
- 某目标没有被探测，是视场、距离、概率、行为时序还是模型 bug，如何逐层证明？
- 新增雷达能力时，哪些证据足以支持“已实现”这一结论？

如果这些问题只能回答术语，说明仍处于工具使用阶段；能够用代码、事件数据和对照实验回答，才进入专家阶段。

## 五、当前项目的准确定位

当前项目已经达到：

- 离线 AI + AFSIM 的工程原型骨架；
- EOIR 想定的结构化生成、校验、仿真和事件结果分析；
- 可审计 RAG、Agent、MCP、Skill 和评测框架；
- 明确的生产模型门禁和能力边界。
- 不依赖大模型的统一核心质量门禁。

当前项目尚未达到：

- 多领域通用 AFSIM 智能想定生成；
- 大规模随机实验与统计推断；
- 真实组织的安全、业务和发布认证；
- 经过大量独立课题验证的专家知识库。

把这四项缺口逐项补齐，才是从“完成教程”到“成为专家”的真正路径。
